# 久久小镇 UX 减噪优化 P0

Issue: #38

## 背景

当前小镇首屏同时呈现品牌、地图、今日指标、居民状态、日志和市政厅入口。信息都有效，但视觉权重接近，用户容易眼花，也不容易判断“我现在该干嘛”。

本轮不做大重构、不新增复杂系统、不动后端，只在 `town.html` 内完成首屏体验收敛。

## 设计原则

1. **主行动先行**：首屏必须有一个由北北给出的明确下一步，而不是只展示数据。
2. **指标降权**：今日指标用于理解状态，不与主行动抢注意力，也不变成一组等价入口。
3. **居民 dock 降噪**：居民 dock 只承载“谁在/是否需要反馈/是否工作中”，详细任务进入市政厅。
4. **地图可理解**：建筑 hover 时显示标签，降低新用户识别成本。
5. **提示不发散**：市政厅、待评价仍保留空间入口和上下文入口，不恢复顶部/指标直达入口。

## P0 改动范围

- 北北提醒卡升级为首屏主行动卡：一句判断 + 一个主按钮。
- 今日小镇指标改为弱化摘要，保留状态展示，不新增点击入口。
- 居民 dock 压缩为头像、姓名、短状态；隐藏积分和最近任务噪声。
- 建筑 hover 显示可见标签与动作提示。
- 市政厅/待评价视觉提示集中到北北主行动、建筑发光和待办上下文按钮。

## 验收标准

- `town.html` 包含 `guardian-primary-action` 主按钮，依据待评分/待批/排队/默认状态切换目标。
- `town-board` 包含 `town-board-subtle` 降权样式，指标仍是 `.town-stat`，不是按钮。
- `resident-chip` 宽度不超过 104px；dock 中不渲染“积分/最近任务”。
- 存在 `building-tip` DOM，并在 canvas hover 时更新显示建筑名和动作提示。
- 保留地图市政厅入口与提醒上下文入口；不恢复顶部“进入市政厅/发布悬赏/评价交付”等重复入口。
- `python tests/test_all_features.py` 通过。
- 浏览器实测：首屏能看到主行动按钮，console 无新增 JS error。
