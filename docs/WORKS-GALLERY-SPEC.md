# 作品长廊 v0.1 页面规格

版本：v0.1
日期：2026-07-04
状态：Draft

---

## 1. 目标

作品长廊是久久光合社的第一门面。它负责把久久和 AI 居民共同完成的优秀产出，以策展方式展示给访客。

v0.1 目标：

- 展示久久手选的 13 个首批精选作品。
- 按内容类型清晰分类。
- 每个作品能打开预览。
- 每个作品能看见创作居民、标签和策展备注。
- 为后续天机阁、久久小镇、资产库建立统一引用关系。

v0.1 不追求：

- 访客直接调用居民。
- 复杂登录和权限系统。
- 自动生成所有缩略图。
- 完整游戏化小镇界面。

---

## 2. 数据来源

作品长廊 v0.1 使用：

```text
docs/curation/featured-works.json
```

每个作品包含：

- `id`
- `title`
- `file_path`
- `work_type`
- `primary_resident`
- `sections`
- `tags`
- `curator_note`

v0.1 先把这个 JSON 当作作品长廊数据源。后续再迁移进数据库的 `works`、`assets`、`asset_links` 表。

## 4. 分类规则

作品长廊使用两级分类。一级分类固定，服务访客快速理解内容形态；二级分类不提前锁死，跟随真实作品生长，由北北或久久策展命名。

不作为一级分类的维度：

- 居民：阿画、小文、北北等作为筛选条件。
- skill：具体 skill 作为标签或天机阁关联。
- 板块：天机阁、久久小镇等作为关联入口。

二级分类示例：

| 一级分类 | 二级分类示例 | 说明 |
|---|---|---|
| 图片 | 猫是命 | 猫主题海报、漫画、视觉作品 |
| 图片 | 小红书封面 | 小红书/社媒封面图 |
| 图片 | 信息图 | 结构化视觉说明 |
| 文章 | AI | AI 使用经验、教程、方法论 |
| 文章 | 成长 | 个人成长、创作复盘、长期记录 |
| 文章 | 教程 | 可被复用的方法文章 |
| 小镇生活 | 小镇晨报 | 小镇早间/晨间内容 |
| 小镇生活 | 小镇晚报 | 小镇晚间内容 |
| 小镇生活 | 奏折 | 北北呈报给久久的治理建议 |
| 小镇生活 | 自评 | 居民或小镇对自身作品/状态的评价 |
| 小镇生活 | 产品体验报告 | 小镇产品巡检和体验报告 |

规则：

- 二级分类可以有诗意和世界观，不要求像数据库枚举一样冷冰冰。
- 二级分类需要来自真实作品，不为空造分类。
- 一个作品只有一个主二级分类，可以通过 tags 承载其他主题。

## 5. 页面规格

### 5.1 长廊首页

页面目标：

- 让访客快速理解“这里不是普通作品集，而是 AI 小镇的策展长廊”。
- 展示首批精选作品。
- 引导进入具体作品和小镇。

首屏内容：

```text
作品长廊
久久和 AI 居民共同创作的第一批作品。
有些是海报，有些是故事，有些只是可爱的生活痕迹。
```

首屏组件：

- 标题：作品长廊。
- 副标题：久久和 AI 居民共同创作的第一批作品。
- 统计：
  - 精选作品 13 件。
  - 居民 5 位核心居民 + 早期扩展居民。
  - 类型 6 类。
- 入口按钮：
  - 进入久久小镇。
  - 查看天机阁。

### 5.2 分类筛选

筛选项：

- 全部
- 图片
- 视频
- 文章
- 小镇生活

二级筛选项随真实作品动态生成。v0.1 只展示当前精选作品中已有的二级分类，不展示空分类。

交互规则：

- 默认显示全部。
- 点击分类后，只显示该分类作品。
- 筛选不改变 URL 的第一版也可以接受。
- 后续可支持 URL query：`?type=comic`。

### 5.3 作品卡片

每张卡片展示：

| 字段 | 说明 |
|---|---|
| 标题 | `title` |
| 类型 | 根据 `work_type` 转中文 |
| 创作居民 | `primary_resident` |
| 标签 | 展示前 3 个 tag |
| 策展短句 | `curator_note` 截断展示 |
| 入口 | 查看作品 |

卡片视觉建议：

- 不做花哨卡片墙。
- 保持策展馆感：大留白、清晰标题、适度缩略图。
- 对 HTML 作品，v0.1 可用 iframe 截图占位或纯文字卡片。
- 卡片圆角不超过 8px。

### 5.4 作品详情页

详情页目标：

- 让访客完整看见作品。
- 让访客理解作品来自哪个居民、哪个方法、哪段小镇生活。

详情页结构：

```text
作品标题
类型 / 创作居民 / 标签
策展说明
作品预览
创作背景
关联入口
  - 去看居民
  - 去天机阁看相关方法
  - 去久久小镇看生活流
```

v0.1 预览规则：

- `.html` 文件：iframe 嵌入预览。
- `.md` 文件：渲染为正文或先用纯文本展示。
- 不存在的文件：展示“作品文件暂不可预览”。

### 5.5 空状态

如果某个分类没有作品：

```text
这一栏还在生长中。
等居民们交付新的作品，北北会把它们送进长廊。
```

## 7. 和其他模块的关系

### 7.1 与资产库

作品长廊不保存源文件。

```text
资产库保存文件和元数据
作品长廊保存策展视图
```

### 7.2 与天机阁

如果作品沉淀了方法，天机阁展示方法卡片，作品长廊展示作品本身。

例：

```text
小文写作检查清单
  作品长廊：作为一篇可读作品展示
  天机阁：提炼为“小文日常感写作法”
```

### 7.3 与久久小镇

久久小镇展示作品产生的过程：

- 谁接了任务。
- 谁完成了。
- 谁被评价。
- 谁因此获得 skill 奖励。

作品长廊展示结果，久久小镇展示过程。

### 7.4 与意图工坊

v0.1 不开放“我也想要”，但作品详情页可预留按钮：

```text
我也想要类似作品
```

按钮状态：

```text
即将开放
```

## 8. 验收标准

- AC1：作品长廊能读取 `featured-works.json`。
- AC2：页面展示 13 个精选作品。
- AC3：作品可按 4 个一级分类和对应二级分类筛选。
- AC4：每张卡片展示标题、类型、居民、标签、策展备注。
- AC5：点击作品能进入详情页。
- AC6：详情页能预览本地 HTML 文件。
- AC7：文件不存在时页面不崩溃。
- AC8：作品长廊不复制作品文件，只引用资产路径。
